Before this, I spent a year at NXP Semiconductors working hands-on with i.MX boards across multiple SoCs: platform bring-up, Wi-Fi driver debugging, and CI/CD automation for the driver build flow.

I hold an M.Tech in Computer Science Engineering from Nirma University (8.36/10) and a B.E. in Information Technology from Gujarat Technological University (8.06/10). What I enjoy most is the detective work of verification tracing a scoreboard mismatch back through waveforms to the exact RTL line that caused it.

Inside the testbench

Tags are cheap, so here is real code. An excerpt from the ALU coverage collector: opcode coverpoints guarded by ALU mode so arithmetic and logical commands are binned separately, value-partitioned operand bins to force zero and boundary stimulus, and ignore_bins to keep the coverage space honest instead of padding the percentage.

testbench/alu_cov.sv · excerpt view full file on GitHub 
covergroup alu_cg;

  arith_cmd_cov: coverpoint cmd iff (mode == 1) {
    bins op_add        = {4'd0};
    bins op_sub        = {4'd1};
    bins op_add_cin    = {4'd2};
    bins op_cmp        = {4'd8};
    bins op_mul_shift  = {4'd10};
    // ... 6 more arithmetic opcodes
    ignore_bins unused = {[11:15]};
  }

  logic_cmd_cov: coverpoint cmd iff (mode == 0) {
    bins op_and        = {4'd0};
    bins op_ror        = {4'd13};
    // ... 12 more logical opcodes
    ignore_bins unused = {[14:15]};
  }

  opa_value_cov: coverpoint opa {
    bins opa_zero  = {0};
    bins opa_small = {[1:127]};
    bins opa_large = {[128:255]};
  }

  overflow_cov: coverpoint oflow {
    bins oflow_set   = {1};
    bins oflow_clear = {0};
  }

endgroup
